# Task: Web Worker wrapping @mlc-ai/web-llm via Comlink

## Description

Build the worker entry (compiled to the separate `dist/worker` chunk from task 001) that instantiates `@mlc-ai/web-llm`'s engine and exposes it across the `postMessage` boundary via `comlink`'s `Comlink.expose`. This worker is the only place the raw `web-llm` engine is ever touched — everything else in the package talks to it through the Comlink proxy.

## Acceptance Criteria
- [ ] Worker module exposes engine methods (load model, generate, streamGenerate token callback, abort) via `Comlink.expose`
- [ ] Main-thread helper wraps `Comlink.wrap<EngineAPI>()` and returns a typed proxy
- [ ] Worker never runs on the main thread — verified by an integration test that asserts the worker file is loaded via `new Worker(...)`, not imported directly
- [ ] Progress callbacks from `web-llm`'s model loader are forwarded across the Comlink boundary as a subscribable event, not a one-shot return value

## Technical Details

This is the core of Stack A (MLC-first + Comlink RPC). Model-loading progress must stream back to the main thread incrementally (task 004 consumes this), not just resolve once at 100%.

## Dependencies

Depends on task 001 (build tooling — needs the worker build chunk to exist).
