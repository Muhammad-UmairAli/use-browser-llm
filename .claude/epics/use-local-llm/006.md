# Task: Cache-status exposure

## Description

Surface whether the requested model is already cached in IndexedDB (via `web-llm`'s own cache) before/while loading, so consumers can distinguish "first-time multi-GB download" from "instant load from cache" in their own UI without this package shipping any UI itself.

## Acceptance Criteria
- [ ] Hook exposes a `cacheStatus` (or equivalent) value distinguishing at least "cached" vs "downloading"
- [ ] Value is available before `status` reaches `ready`, so a consumer can show a different message for a fresh download vs a cache hit
- [ ] No new cache mechanism is built — this task only surfaces what `web-llm` already tracks internally, it does not duplicate or replace it

## Technical Details

Reads cache-hit information from the worker (task 003) alongside the existing progress stream; extends task 004's state machine rather than introducing a separate one.

## Dependencies

Depends on task 003 (worker boundary) and task 004 (state machine to extend).
