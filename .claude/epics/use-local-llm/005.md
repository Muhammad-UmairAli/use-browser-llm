# Task: Hook generate/streamGenerate API + cancellation

## Description

Add the generation half of the public hook API on top of the load-state machine from task 004: an awaited `generate(prompt, opts)` returning the full completion, a `streamGenerate(prompt, opts)` yielding tokens incrementally, and an abort mechanism so a consumer can cancel an in-flight generation from the UI.

## Acceptance Criteria
- [ ] `generate()` resolves with the full completion text once generation finishes
- [ ] `streamGenerate()` yields tokens incrementally (async iterator or callback — pick one and document it)
- [ ] Calling the returned abort function stops generation and the worker's inference loop, not just the UI-visible promise
- [ ] Calling `generate`/`streamGenerate` before `status === 'ready'` rejects with a typed error rather than hanging or throwing an unhandled exception

## Technical Details

Both methods proxy through the same Comlink-wrapped engine from task 003; `streamGenerate`'s token stream is forwarded the same way load-progress events are (task 003's progress-forwarding pattern).

## Dependencies

Depends on task 004 (needs a `ready` state to generate against).
