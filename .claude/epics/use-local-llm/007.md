# Task: Error handling & unsupported-browser fallback path

## Description

Wire the WebGPU capability check (task 002) into the hook's state machine (task 004) so unsupported browsers/devices land in a distinct `unsupported` status with a machine-readable reason, and harden all other failure paths (model-load failure, worker crash, out-of-memory) into typed, catchable errors instead of unhandled exceptions or a frozen tab.

## Acceptance Criteria
- [ ] Capability check runs before any model-load attempt; failure short-circuits straight to `unsupported`, never `loading`
- [ ] Worker crash / unexpected termination surfaces as a typed `error`, and the hook remains usable afterward (a fresh model request can recover)
- [ ] Out-of-memory / model-too-large failure from the engine is caught and reported, not left as an unhandled worker exception
- [ ] Every error path is covered by a test that asserts the specific typed error/state, not just "it doesn't crash"

## Technical Details

Extends task 004's state machine; consumes task 002's detection utility directly (no need to go through the worker for the capability check itself, since it runs on the main thread).

## Dependencies

Depends on task 002 (capability detection) and task 004 (state machine to extend).
