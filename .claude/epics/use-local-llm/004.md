# Task: Hook state machine for model loading

## Description

Implement the `useLocalLLM()` hook's core state machine covering `idle` → `loading` → `ready` / `error` / `unsupported`, wired to the Comlink-wrapped worker proxy from task 003 and the progress events it forwards. This is the load-time half of the public hook API; task 005 adds the generate-time half on top of it.

## Acceptance Criteria
- [ ] `status` transitions correctly: `idle` on mount (before any model requested), `loading` once a model id is passed, `ready` once the worker confirms load, `error` on load failure
- [ ] `progress` updates incrementally (0–1) as the worker forwards download progress, not just a final jump to 1
- [ ] State updates are safe under React 18 concurrent rendering (no state updates after unmount, no tearing)
- [ ] Re-requesting a different model id while one is already loaded cancels/replaces the in-flight load cleanly

## Technical Details

Consumes the Comlink proxy and progress event stream from task 003. Does not yet handle the `unsupported` path (task 007) or generation (task 005) — scope this task to load-state only.

## Dependencies

Depends on task 003 (worker + Comlink boundary must exist to wire against).
